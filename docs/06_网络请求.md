# 第6章：Combine 与网络请求

> 对应代码：`Chapter6_Networking/NetworkingViewController.swift`

网络请求是 Combine 最常见的实际应用场景。`URLSession` 原生提供了 `dataTaskPublisher` 方法，将网络请求融入 Combine 管道。

---

## 6.1 dataTaskPublisher 基础

```swift
let url = URL(string: "https://api.example.com/posts/1")!

URLSession.shared.dataTaskPublisher(for: url)
    // 类型: AnyPublisher<(data: Data, response: URLResponse), URLError>
```

返回的 Publisher：
- **Output**：`(data: Data, response: URLResponse)` 元组
- **Failure**：`URLError`

---

## 6.2 典型网络请求管道

### 步骤分解

```swift
URLSession.shared.dataTaskPublisher(for: url)   // 1. 发起请求
    .map(\.data)                                  // 2. 提取 Data
    .decode(type: Post.self, decoder: JSONDecoder()) // 3. JSON → Model
    .receive(on: DispatchQueue.main)              // 4. 切到主线程
    .sink(                                        // 5. 处理结果
        receiveCompletion: { completion in
            switch completion {
            case .finished: break
            case .failure(let error): handleError(error)
            }
        },
        receiveValue: { post in
            updateUI(with: post)
        }
    )
    .store(in: &cancellables)                     // 6. 保存订阅
```

### 每一步的类型变化

```
dataTaskPublisher  → Publisher<(Data, URLResponse), URLError>
  .map(\.data)     → Publisher<Data, URLError>
  .decode(...)     → Publisher<Post, Error>          ← 注意: Failure 变成 Error
  .receive(on:)    → Publisher<Post, Error>
  .sink(...)       → AnyCancellable
```

---

## 6.3 JSON 解析

### decode 操作符

`decode(type:decoder:)` 将 `Data` 解析为 `Decodable` 类型：

```swift
struct Post: Codable {
    let id: Int
    let title: String
    let body: String
}

.decode(type: Post.self, decoder: JSONDecoder())
```

**注意**：`decode` 会将 Failure 类型从 `URLError` 提升为 `Error`（因为解码也可能失败）。

### 自定义 JSONDecoder

```swift
let decoder = JSONDecoder()
decoder.keyDecodingStrategy = .convertFromSnakeCase  // snake_case → camelCase
decoder.dateDecodingStrategy = .iso8601              // ISO 8601 日期格式

.decode(type: Post.self, decoder: decoder)
```

## 6.4 错误处理策略

### 策略一：retry + catch（推荐）

```swift
URLSession.shared.dataTaskPublisher(for: url)
    .map(\.data)
    .decode(type: Post.self, decoder: JSONDecoder())
    .retry(2)                              // 失败后重试2次
    .catch { error -> Just<Post> in        // 全部失败后用默认值
        Just(Post.default)
    }
    .receive(on: DispatchQueue.main)
    .sink { post in updateUI(with: post) }
```

### 策略二：replaceError（简洁版）

```swift
.replaceError(with: Post.default)  // 一行搞定错误处理
```

### 策略三：mapError + 统一错误类型

```swift
enum AppError: Error {
    case network(String)
    case decode(String)
    case unknown
}

URLSession.shared.dataTaskPublisher(for: url)
    .mapError { AppError.network($0.localizedDescription) }
    .map(\.data)
    .decode(type: Post.self, decoder: JSONDecoder())
    .mapError { error -> AppError in
        if error is DecodingError {
            return .decode(error.localizedDescription)
        }
        return error as? AppError ?? .unknown
    }
```

## 6.5 链式请求（flatMap）

当第二个请求依赖第一个请求的结果时，使用 `flatMap`：

```swift
// 1. 获取 Post → 2. 用 Post 的 userId 获取用户信息
URLSession.shared.dataTaskPublisher(for: postURL)
    .map(\.data)
    .decode(type: Post.self, decoder: JSONDecoder())
    .flatMap { post -> AnyPublisher<User, Error> in
        let userURL = URL(string: "https://api.example.com/users/\(post.userId)")!
        return URLSession.shared.dataTaskPublisher(for: userURL)
            .map(\.data)
            .decode(type: User.self, decoder: JSONDecoder())
            .eraseToAnyPublisher()
    }
    .receive(on: DispatchQueue.main)
    .sink(...)
```

**对比回调地狱**：

```swift
// ❌ 传统嵌套回调
fetchPost { post in
    fetchUser(id: post.userId) { user in
        fetchAvatar(url: user.avatarURL) { image in
            // 三层嵌套...
        }
    }
}

// ✅ Combine 扁平链式调用
fetchPostPublisher()
    .flatMap { post in fetchUserPublisher(id: post.userId) }
    .flatMap { user in fetchAvatarPublisher(url: user.avatarURL) }
    .receive(on: DispatchQueue.main)
    .sink { image in ... }
```

## 6.6 并行请求

当多个请求**互不依赖**时，使用 `combineLatest` 或 `zip` 并行执行：

```swift
let postsPublisher = URLSession.shared.dataTaskPublisher(for: postsURL)
    .map(\.data).decode(type: [Post].self, decoder: JSONDecoder())

let usersPublisher = URLSession.shared.dataTaskPublisher(for: usersURL)
    .map(\.data).decode(type: [User].self, decoder: JSONDecoder())

// 两个请求并行执行，都完成后才触发 sink
postsPublisher.zip(usersPublisher)
    .receive(on: DispatchQueue.main)
    .sink(
        receiveCompletion: { ... },
        receiveValue: { posts, users in
            // 两个请求都完成了
        }
    )
```

## 6.7 请求取消

Combine 的订阅取消机制天然支持请求取消：

```swift
// 存储订阅
var searchCancellable: AnyCancellable?

func search(query: String) {
    searchCancellable?.cancel()  // 取消之前的请求
    searchCancellable = URLSession.shared.dataTaskPublisher(for: makeURL(query))
        .map(\.data)
        .decode(type: SearchResult.self, decoder: JSONDecoder())
        .receive(on: DispatchQueue.main)
        .sink(...)
}
```

或者用 `switchToLatest` 自动取消旧请求：

```swift
$searchText
    .debounce(for: .milliseconds(300), scheduler: DispatchQueue.main)
    .map { query in
        URLSession.shared.dataTaskPublisher(for: makeURL(query))
            .map(\.data)
            .decode(type: SearchResult.self, decoder: JSONDecoder())
            .catch { _ in Empty() }
    }
    .switchToLatest()  // 自动取消旧的网络请求
    .receive(on: DispatchQueue.main)
    .sink { result in ... }
```
